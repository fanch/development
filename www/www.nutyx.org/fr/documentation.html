Documentation

Petit lexique du NuTyXien Tous les termes utilisés dans la documentation sont référencés et expliqués dans ce petit lexique
Installation de base classique sur une seule partition en mode BIOS L'installation proposée ici consiste à utiliser un disque complet, une seule partition et l'outil d'amorçage (GRUB) installé sur la même partition. Pour ceux qui souhaitent une première approche de l'installation sans se prendre la tête.
Installation de base classique sur une seule partition en mode UEFI L'installation proposée ici consiste à utiliser un disque complet, une seule partition sans outil d'amorçage (GRUB). Cette methode peut-être utilisée si votre machine supporte l'UEFI, vous pouvez alors tirer profit de cette fonctionnalitée. Pour ceux qui souhaitent une approche de l'installation en mod UEFI isans se prendre la tête.
Finalisation de votre installation. Mise en place d'une interface graphique et quelques outils bien utiles.
Les commandes de base Il n'y a pas encore de gestionnaire de paquets graphique, ces quelques commandes de base vont vous permettre de vous débrouiller très rapidement avec NuTyX
Installation de NuTyX depuis le script sans connection internet L'installation proposée ici consiste à utiliser le script d'installation disponible en téléchargement Cette approche permet d'installer NuTyX depuis une autre NuTyX ou même une autre distribution. L'article explique comment installer NuTyX sans connection internet.
Mise en place de GRUB Si vous n'avez pas installé de procédure d'amorçage (boot loader) lors de l'installation de votre NuTyX (par erreur ou par volonté), il n'est pas encore trop tard. Avec l'ISO c'est très simple d'installer GRUB sur votre machine et de façon indépendante. Cet article vous montre la procédure en 3 ou 4 écrans.
Manuel de NuTyX de la version 9.0 Manuel de NuTyX version 9.0. Vous y trouverez tout ce qu'il faut savoir pour la configuration et l'utiliisation de votre NuTyX
Mise en place d'un serveur HTTP Ce petit guide explique la mise en place d'un serveur HTTP sur NuTyX. Un serveur http peut s'avérer utile si l'on souhaite par exemple tester des pages html sur un vrai serveur. Si l'on souhaite créer son propre site web familial uniquement accessible depuis la maison ou, pourquoi pas, le rendre accessible sur la toile.
Mise en place d'un serveur NFS Ce petit guide explique la mise en place d'un serveur NFS sur NuTyX. NFS vous permet de partager des fichiers entre plusieurs machines connectées entre elle au sein d'un réseau familial. La seule condition, toutes les machines connectées doivent supporter le service NFS. Heureusement pour nous toutes les distributions le supportent.
Mise en place d'un parefeu simple avec Iptables Ce petit guide explique la mise en place d'un pare-feu "de base" sur NuTyX. Il devrait vous permettre d'avoir un pare-feu qui démarre et s'éteint automatiquement avec votre réseau, tout en protégeant votre machine.
Construire son propre noyau Il existe beaucoup de raisons de vouloir compiler son propre noyau. Avoir un noyau optimalisé pour son propre matériel est sans doute la meilleure raison. Cela permet d'avoir un noyau optimalisé en taille, en vitesse de lancement, en utilisation de la charge du processeur, etc. Si vous avez choisi de compiler et installer votre propre noyau, suivez pas à pas cet article.
Produire une ISO installable Il est possible de générer très facilement une ISO depuis une NuTyX installée dans un dossier. Cette ISO peut ensuite être réinstallée et servir de base de déploiement. Libre à vous d'ajouter tous les logiciels que vous souhaitez, y compris une interface graphique. Le minimum à ajouter est un kernel.
Pourquoi contribuer ? NuTyX dispose actuellement de quelques 1400 applications disponibles en binaires. C'est à la fois beaucoup et pas beaucoup. En effet tous les jours ou presque, sortent de nouvelles applications sous licence libre (ou pas)
Quelles sont les connaissances requises ? Pour pouvoir contribuer, il ne faut pas être un programmeur averti. Quelques notions rudimentaires du langage BASH suffisent. Si vous n'avez aucune idée du langage BASH, cet article vous expliquera les notions de base pour réaliser une recette sans se prendre la tête
Qu'est-ce qu'un port, une recette) ? Pour constuire votre premier paquet, vous avez besoin de créer une recette. Cette recette se trouve dans un port qui est un dossier contenant un certain nombre de fichiers nécessaire pour la construction d'un paquet. La commande pkgmk permet de "construire" le paquet qui sera ensuite installable.
Comment construire un paquet ? Cette section vous explique comment construire depuis zéro un paquet pour NuTyX. On y aborde toutes les fonctionnalités (de base et avancées) du gestionnaire de paquets cards.
Comment construire dans une chroot ? Cet article explique comment construire vos paquets dans un environement isolé. Vous DEVEZ utiliser cette méthode si vous n'êtes pas sur un système de base ou si vous avez choisi d'utiliser NuTyX comme une distribution binaire comme expliqué dans la FAQ.
Construire sa NuTyX Cette section vous explique comment construire depuis zéro une distribution linux, en l'occurance une NuTyX en se basant sur l'excellent manuel en ligne de LFS se trouvant à l'adresse http://www.linuxfromscratch.or...ew/development/.
Cet article s'adresse à un public maîtrisant sa distribution GNU/linux. Il est d'ailleurs vivement conseillé d'avoir construit une première fois une LFS à la main. Ce projet étant en perpétuel évolution, il est posssible que vous rencontreriez des erreurs lors de sa construction.
L'article sera divisé en quatres sections:
  1. La préparation de la construction
  2. La construction de la CHROOT (pass 1)
  3. La construction de la NuTyX (pass 2)
  4. La post-construction et ses options
Mise en place d'un serveur de compilation Cet article s'adresse principalement aux empaqueteurs de NuTyX. Il réunit toutes les techniques décrites plus haut et permet la construction de tous les ports d'une collection à l'aide d'une simple commande: 
build-collection /development/kde5
Il est indispensable de maîtriser sa NuTyX pour mener à bien ce tutoriel..
