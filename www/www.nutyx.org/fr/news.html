NuTyX 9.0 disponible avec cards 2.2

L'équipe de NuTyX est très fière de vous proposer la version NuTyX 9.0.

NuTyX 9.0 propose le kernel 4.10.8, kernel lts 4.9.20, glibc 2.25, gcc 6.3.0, binutils 2.28, python 3.6.0, xorg-server 1.19.2, qt 5.8.0, plasma 5.9.4, kf5 5.31.0, gnome 3.22.2, mate 1.16.1, xfce4 4.12.3, firefox 52.0.2 et beaucoup d'autres mise à jour.

Plus de 2500 commits depuis la version 8.2

Des nouvelles isos sont disponible en 32 et 64 bits. Leur tailles sont respectivement 246 MB et 247 MB sur http://downloads.nutyx.org.

Les interfaces graphiques disponibles sont: kde5, gnome, mate, xfce4, lxde, flwm, jwm, ratpoison, blackbox, fluxbox, openbox, bspwm, icewm, twm, etc.

Mise à jour majeure

Cette nouvelle version est le résultat de plus de 6 mois de dévelopement. Une mise à jour la 8.2.1 est peut-être possible, mais risquée, l'équipe de NuTyX recommande de faire une nouvelle installation.

Nouveautés dans les ISOs

Ces nouvelles isos peuvent être lancées sur des machines configurées en mode UEFI. Si tel est le cas, l'installateur aide l'utilisateur à faire les bons choix lors de l'installation et l'uefi se chargera de l'amorçage de la machine, aucun gestionnaire d'amorçage tiers n'est utilisé par défaut. Libre à l'utilisateur d'en installer un si il le souhaite

Vous trouverez un tutoriel, comment installer NuTyX en mode UEFI dans la document du site et sur la chaîne YouTube.

L'ISO peut se charger à 100 % en ram si la machine cible dispose plus qu'1Gb de ram.. La clé USB peut donc être déconnectée et ainsi libérer le port USB. Utile dans le cas de l'installation sur tablette ne disposant que d'un seul port USB

Nouveau kernel LTS

Le support à long terme du kernel est passé de la version 4.4 -> 4.9.

Plus d'initrd par défaut

L'initrd étant devenue obsolète, il n'est plus necessaire de spécifier la ligne initrd dans le fichier d'amorçage grub. les systèmes de fichiers reconnues par NuTyX sont compilés dans le kernel.

Les composants principaux de la chaîne de compilation sont mis à jour.

Les composants principaux de la chaîne d'outils cad en-têtes du noyau, compilateur, editeur de liens, assembleur et librairie C sont désormais à jour, les versions respectives de chaque composant se trouvent dans la liste ci-dessous.

Collection base-extra supprimée

La collection base-extra est désormais intégrée dans la collection base.

Le nombre de collections officielles est donc désormais 13 collections:

  • base
  • cli, cli-extra
  • gui, gui-extra
  • xfce4, xfce4-extra
  • mate, mate-extra
  • gnome, gnome-extra
  • kde5, kde5-extra

Renommage et Splitting

Le renommage de python3 en python, le split de qt5 et les dernières mise à jour ont eu de grosses répercussions sur les paquets qui en dépendent et ont entrainé plus de travail que prévu. C'est pourquoi l'équipe de NuTyX a décidé de recompiler toutes les collections.

  • python -> python2
  • python3 -> python
  • qt5 -> qt
  • fltk1 -> fltk
  • xorg-mesa -> mesa

Cards 2.2

Il s'agit d'une mise à jour majeure. Cards est intégralement traduit en français.

Cards affiche plus de résultats lors d'une recherche de paquets. Si le paquet trouvé est déjà installé, celui-ci s'affichera en vert dans le résultat de la recherche. Désormais, celle-ci s'effectue également dans les alias et les groupes.

Structure des dossiers simplifiée

Les liens /usr/lib64 -> /usr/lib et /lib64 -> /lib sont supprimés.

Le dossier /usr/lib64 ne doit plus être présent.

Le dossier /lib64 ne doit contenir que 2 liens vers le même fichier /lib/ld-linux-x86-64.so.2

Le wifi

Pour la gestion du wifi, c'est désormais le paquet networkmanager qui est préféré. Sa grande souplesse permet de configurer votre wifi très facilement. L'outil wicd basé sur python est toujours disponible comme alternative pour se connecter en wifi.

La post-configuration du wifi a été amélioré. Le bug de l'ESSID corrigé. Un nom contenant des espaces est désormais possible.

La chaîne YouTube

https://www.youtube.com/channel/UChR816uG9bS4eOQfKY-xB8g

Une nouvelle chaîne YouTube est ouverte. Elle contient divers tutoriels disponibles sur le site.

Plusieurs fois demandées, vous pouvez désormais visionnez une installation en UEFI ou en bios.

Paquets NuTyX 8.2 déplacés

Si vous êtes actuellement sur la dernière version de NuTyX 8, et si vous n'êtes pas encore disposé à passer sur la NuTyX 9.0, c'est tout à fait possible de continuer à utiliser votre version 8

Vous devez MAINTENANT effectuer une petite manipulation dans votre fichier /etc/cards.conf. 

sudo sed -i s@downloads.nutyx.org@downloads.tuxfamily.org/nutyx@ /etc/cards.conf
